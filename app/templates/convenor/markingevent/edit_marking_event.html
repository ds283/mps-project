Targets
Grade target vocabulary

Each key in the JSON object is a target name (a valid Python identifier); its value is a Python expression that computes the final grade for that target by combining grades from the constituent marking workflows.

Three target names are reserved and cause grades to be propagated to the submission record when you use the corresponding “push grade” action:

supervisor
Pushes to supervision_grade (requires supervision grade enabled on the period)
report
Pushes to report_grade (requires report grade enabled on the period)
presentation
Pushes to presentation_grade (requires presentation grade enabled on the period)

Expression variables — reference each marking workflow by its key field. At conflation time each variable evaluates to a float in the range 0–100, equal to that workflow’s submitted grade. Keys must match exactly.

Available functions — abs, min, max, round, sum. No other built-ins are accessible.

Example (workflows keyed supervisor_wf, marker_a, marker_b):

{
  "supervisor": "supervisor_wf",
  "report": "marker_a * 0.5 + marker_b * 0.5"
}
